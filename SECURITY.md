# Security Policy

## Supported Versions

This project is still early-stage and does not yet guarantee long-term support
windows for older releases.

At the moment, the latest published version should be treated as the supported
line.

## Reporting A Vulnerability

Until the public GitHub repository and maintainer contact path are finalized,
please do not post potential security issues publicly in GitHub issues.

Once the repository is public, add one private reporting path here:

- GitHub private vulnerability reporting
- maintainer security email
- organization security contact

## Scope

Examples of issues worth reporting privately:

- memory safety issues in unsafe code, if any is added later
- logic bugs that could lead to incorrect or dangerous engineering outputs in
  downstream automated workflows
- dependency or release process compromise

At the current stage, most likely risks are release-process and dependency
related rather than classic application attack surface issues.
