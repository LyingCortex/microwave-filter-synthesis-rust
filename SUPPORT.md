# Support

## Getting Help

For now, the best support paths are:

- open a GitHub issue for bugs or documentation problems
- open a feature request for API or algorithm suggestions
- read the docs in `docs/` for architecture and roadmap context first

## Good Issue Reports

A helpful report usually includes:

- the exact API call or example used
- input specification values
- expected behavior
- actual behavior
- Rust version and platform

## Project Scope

`mfs` is a core library project. Support is strongest for:

- API behavior
- numerical correctness issues
- architecture and contribution questions
- benchmark and regression fixture discussion

Support is weaker for:

- GUI workflows
- plotting frontends
- application-specific deployment setup
